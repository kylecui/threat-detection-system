## 📝 注意事项

### 补充原则

1. **保持一致性**: 所有文档遵循统一格式
2. **实用为主**: 代码示例必须可运行
3. **场景驱动**: 提供真实的使用场景
4. **故障友好**: 详细的排查步骤
5. **持续更新**: API变更时同步更新

### 常见误区

❌ **避免**:
- 复制粘贴未验证的代码
- 使用过时的API端点
- 忽略错误处理
- 缺少响应示例
- 表格格式不统一

✅ **推荐**:
- 所有代码在本地测试通过
- 使用最新的API版本
- 完善的异常处理示例
- 成功和失败响应都提供
- 统一使用Markdown表格语法
