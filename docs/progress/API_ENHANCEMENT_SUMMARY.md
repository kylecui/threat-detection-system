## 📊 标准化模板

### 详细文档应包含的9个章节:

1. **文档头部** - 服务信息、版本、完整目录
2. **系统概述** (~200行) - 核心特性、工作流程、架构组件
3. **API端点详细文档** (每个端点 ~150行) - 参数、示例、响应、错误码
4. **数据模型** (~100行) - DTO定义、字段说明、JSON Schema
5. **使用场景** (~300行) - 至少3个真实场景 + 完整代码
6. **Java客户端示例** (~200行) - 完整客户端类 + 使用示例
7. **最佳实践** (~150行) - 推荐做法、避免事项、优化建议
8. **故障排查** (~200行) - 常见问题、排查步骤、解决方案
9. **附录** (~100行) - 配置参考、版本历史、相关链接

**curl示例**:
```bash

  }'
```
