## 🎯 核心发现

### 发现1: 蜜罐系统的独特性

| 维度 | 传统IDS/IPS | 蜜罐系统 |
|------|------------|---------|
| **部署位置** | 网络边界 | **内网二层** |
| **检测对象** | 外部攻击 | **内部威胁/横向移动** |
| **误报率** | 中-高 | **极低** (所有访问都是异常) |
| **攻击阶段** | 初始访问 | **横向移动/权限提升** |
| **典型威胁** | 扫描、DDoS | **APT、勒索软件** |

**关键洞察**: 蜜罐系统检测的是**已进入内网**的威胁,这与传统防御系统完全不同。

### 发现2: 端口 = 攻击意图指纹

**端口选择暴露攻击目标**:

```
攻击者尝试的端口 → 推断攻击意图 → 评估威胁级别

示例:
- 3389 (RDP)       → 远程控制        → 严重威胁 (权重 3.0)
- 445 (SMB)        → 横向移动/勒索   → 严重威胁 (权重 3.0)
- 22 (SSH)         → 远程登录        → 高危威胁 (权重 2.5)
- 3306 (MySQL)     → 数据窃取        → 高危威胁 (权重 2.5)
- 80/443 (HTTP/S)  → Web攻击         → 中等威胁 (权重 1.5)

组合模式:
- 3389 + 445       → APT横向移动     → 极严重 (权重 3.0 × 1.3)
- 5+端口           → 广泛扫描        → 严重 (权重 × 1.3)
```

**原系统的219个端口配置是正确的** - 反映攻击意图严重性,而非端口扫描复杂度。

### 发现3: 多诱饵访问 = 横向移动

**访问模式分析**:

| 诱饵访问数 | 行为特征 | 威胁级别 | 权重 |
|-----------|---------|---------|------|
| 1个 | 零星探测 | INFO/LOW | 1.0 |
| 2-3个 | 小范围扫描 | MEDIUM | 1.5 |
| 4-5个 | 系统性扫描 | HIGH | 2.0 |
| 6-10个 | 横向移动 | HIGH | 2.5 |
| 10+个 | APT级威胁 | CRITICAL | 3.0 |

**典型攻击链**:
```
1. 初始失陷 (钓鱼、0day)
2. 建立据点
3. 内网侦察 → 扫描IP段 → 访问多个诱饵 ← 蜜罐捕获
4. 横向移动
5. 权限提升
6. 达成目标 (数据窃取/勒索)
```

## 🎓 关键教训

### 1. 理解工作原理的重要性

**教训**: 不理解蜜罐机制导致对整个系统的误解,影响评分算法设计。

**启示**: 在系统设计前,必须深入理解核心工作原理。

### 2. 字段语义的重要性

**教训**: `response_ip` 命名导致误解为"被攻击的服务器IP"。

**启示**: 字段命名应明确反映真实语义 (如 `honeypotIp` 更清晰)。

### 3. 威胁模型决定设计

**教训**: 基于"外部攻击"假设设计的评分算法不适用于"内网威胁"。

**启示**: 威胁模型是系统设计的基础,必须准确。
