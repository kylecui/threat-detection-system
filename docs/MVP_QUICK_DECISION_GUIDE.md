## 🚦 3个关键决策点

### 决策点 #1: kSwitch蜜罐 (Week 4)

**决策问题**: 是否实施L1 eBPF蜜罐？

**评估指标**:
| 指标 | Go条件 | No-Go条件 |
|------|--------|-----------|
| **技术可行性** | kSwitch编译成功 | 编译失败 |
| **团队能力** | 有2年eBPF经验 | 无相关经验 |
| **时间成本** | ≤ 4周 | > 6周 |
| **风险** | 可控 | 不可控 |

**Go方案**:
```
Week 5-8: L1 eBPF蜜罐开发
- TCP SYN-ACK响应 + Banner
- 威胁驱动动态配置
- APT覆盖率: 14% → 37%
- 额外成本: $50K
```

**No-Go方案 (降级)**:
```
Week 5-6: L2容器蜜罐 (Dionaea + Cowrie)
- Docker部署
- 日志采集 (Filebeat → Kafka)
- APT覆盖率: 14% → 64% (跳过L1)
- 节省时间: 2周
- 节省成本: $50K
```

**决策建议**: 
- 如果没有eBPF专家 → **No-Go (降级到L2)**
- 如果有专家且时间充足 → **Go**

---

### 决策点 #2: ML模型 (Week 18)

**决策问题**: 是否部署ML误报过滤？

**前置工作** (Week 13-17):
```
- 标注500条样本 (初步验证)
- 训练Random Forest模型
- 测试准确率和误报率下降
- 成本: $8,000 (2人 × 40小时)
```

**评估指标**:
| 指标 | Go条件 | No-Go条件 |
|------|--------|-----------|
| **准确率** | ≥ 90% | < 85% |
| **误报下降** | ≥ 50% | < 30% |
| **标注成本** | 可接受 | 超预算 |

**Go方案**:
```
Week 19-22: 完整ML部署
- 继续标注到2000条
- 完整模型训练
- 生产部署
- 误报率下降: 70%
- 额外成本: $8,700
```

**No-Go方案 (降级)**:
```
Week 19-20: 规则白名单优化
- 更精确的Windows Update特征
- 扩展业务系统白名单
- 时间模式规则
- 误报率下降: 40%
- 节省成本: $8,700
```

**决策建议**: 
- 如果500条样本效果好 (准确率>90%) → **Go**
- 如果效果一般或成本超预算 → **No-Go (规则优化)**

---

### 决策点 #3: APT状态机 (Week 26)

**决策问题**: 是否实施完整6阶段APT状态机？

**前置工作** (Week 23-25):
```
- 实现简化版3阶段状态机
- 红队演练验证
- 性能测试 (Checkpoint时间)
```

**评估指标**:
| 指标 | Go条件 | No-Go条件 |
|------|--------|-----------|
| **APT检出率** | ≥ 80% | < 60% |
| **误报率** | < 5% | > 10% |
| **性能** | Checkpoint < 1min | > 2min |

**Go方案**:
```
Week 27-28: 完整6阶段状态机
- 侦察 → 武器化 → 投递 → 利用 → 安装 → C&C
- 精细化阈值调优
- APT检出率: 95%
```

**No-Go方案 (降级)**:
```
Week 27-28: 持续性评分
- persistenceScore = attackDuration × avgAttackRate
- 不做精细阶段划分
- APT检出率: 70%
- 简化复杂度
```

**决策建议**: 
- 如果红队验证效果好 → **Go (完整状态机)**
- 如果验证效果一般或性能不足 → **No-Go (持续性评分)**
