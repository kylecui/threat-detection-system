## 问题背景

### 发现的问题

在之前的实现中,时间窗口参数 `time_window_seconds` 虽然作为API参数存在,但**并未真正影响威胁评分计算**:

```
相同攻击数据 (30次攻击):
- 30秒窗口 (1次/秒爆发):   评分 79.56 ❌
- 300秒窗口 (0.1次/秒):    评分 79.56 ❌  
- 3600秒窗口 (0.008次/秒): 评分 79.56 ❌

问题: 完全相同的评分,无法区分爆发性攻击和持续性扫描
```

### 不合理的地方

- **30秒内30次攻击** = 高速爆发 (1次/秒) → 应该是高危威胁
- **3600秒内30次攻击** = 慢速扫描 (0.008次/秒) → 应该是低危威胁

但两者得到**完全相同的评分**,这不合理。

---

## 解决方案: 攻击速率权重

### 核心思想

**引入攻击速率 (Attack Rate) 的概念**:

```
攻击速率 = attackCount / timeWindowSeconds (次/秒)
```

基于攻击速率动态调整威胁评分权重:
- **高速率爆发**: 权重提升 1.5-2.5倍
- **中等速率**: 权重保持基准 1.0-1.5倍  
- **低速率持续**: 权重降低 0.5-1.0倍

### 算法实现

```java
/**
 * 计算攻击速率权重 (V5.1新增)
 */
public double calculateAttackRateWeight(int attackCount, Integer timeWindowSeconds) {
    // 默认时间窗口: 300秒 (5分钟)
    int windowSeconds = (timeWindowSeconds != null && timeWindowSeconds > 0) 
                       ? timeWindowSeconds : 300;
    
    // 计算攻击速率 (次/秒)
    double attackRate = (double) attackCount / windowSeconds;
    
    if (attackRate >= 10.0) {
        // 极高速率: >10次/秒 → DDoS级别
        return 2.5;
    } else if (attackRate >= 1.0) {
        // 高速率: 1-10次/秒 → 爆发性攻击
        // 线性映射: 1次/秒=1.5, 10次/秒=2.0
        return 1.5 + (attackRate - 1.0) / 9.0 * 0.5;
    } else if (attackRate >= 0.1) {
        // 中等速率: 0.1-1次/秒 → 标准扫描
        // 线性映射: 0.1次/秒=1.0, 1次/秒=1.5
        return 1.0 + (attackRate - 0.1) / 0.9 * 0.5;
    } else if (attackRate >= 0.01) {
        // 低速率: 0.01-0.1次/秒 → 慢速扫描
        // 线性映射: 0.01次/秒=0.7, 0.1次/秒=1.0
        return 0.7 + (attackRate - 0.01) / 0.09 * 0.3;
    } else {
        // 极低速率: <0.01次/秒 → 超慢速扫描
        return Math.max(0.5, 0.5 + attackRate / 0.01 * 0.2);
    }
}
```

### 权重分段详解

| 攻击速率 | 权重范围 | 攻击特征 | 示例场景 |
|---------|---------|---------|---------|
| **>10次/秒** | 2.5x | DDoS级别爆发 | 100次/10秒 |
| **1-10次/秒** | 1.5-2.0x | 爆发性攻击 | 30次/30秒 |
| **0.1-1次/秒** | 1.0-1.5x | 标准扫描 | 30次/300秒 |
| **0.01-0.1次/秒** | 0.7-1.0x | 慢速扫描 | 30次/900秒 |
| **<0.01次/秒** | 0.5-0.7x | 超慢速扫描 | 30次/3600秒 |

## 代码变更

### 修改的文件

1. **AggregatedAttackData.java**
   - 添加 `timeWindowSeconds` 字段
   - 用于传递时间窗口信息到评分计算器

2. **AssessmentRequest.java**
   - 添加 `time_window_seconds` API参数
   - 在 `toAggregatedData()` 方法中传递该参数

3. **ThreatScoreCalculator.java**
   - 添加 `calculateAttackRateWeight()` 方法
   - 在 `calculateThreatScore()` 中应用攻击速率权重
   - 更新评分公式: `rawScore *= attackRateWeight`

### 核心公式变更

**V5.0 (旧版本)**:
```java
rawScore = baseScore × timeWeight × ipWeight × portWeight × deviceWeight 
         × combinedSegmentWeight
```

**V5.1 (新版本)**:
```java
rawScore = baseScore × timeWeight × ipWeight × portWeight × deviceWeight 
         × combinedSegmentWeight × attackRateWeight  // ✨ 新增
```

## 应用场景

### 场景1: 实时爆发性攻击检测

**需求**: 快速检测和响应突发性攻击行为

**配置**:
```json
{
  "time_window_seconds": 30,  // 30秒窗口
  "alert_threshold": "CRITICAL"
}
```

**效果**: 
- 30秒内30次攻击 → 评分 83.96 (CRITICAL) → 立即告警 ✅
- 触发自动隔离机制

### 场景2: 标准安全监控

**需求**: 平衡准确性和敏感度

**配置**:
```json
{
  "time_window_seconds": 300,  // 5分钟窗口 (默认)
```

**效果**:
- 5分钟内30次攻击 → 评分 79.56 (HIGH) → 标准告警
