````markdown
## 🎯 一句话项目描述

**一个基于云原生技术的内网蜜罐威胁检测系统**，通过在未使用的IP上设置虚拟哨兵（诱饵），主动诱捕并识别内网失陷主机的恶意行为（横向移动、渗透、勒索软件传播等），相比原C#/Windows/MySQL系统，实现了微服务架构、实时流处理、多租户隔离和多通道告警。

## 🏛️ 项目架构五层认知

### 第一层: 业务层 - 蜜罐机制

```
终端设备 (honeypot device)
    ↓ 监控二层网络
识别未使用IP
    ↓ 主动响应ARP/ICMP
诱导攻击者尝试连接
    ↓ 记录攻击者信息
攻击MAC, 攻击IP, 尝试端口, 时间戳
    ↓ 判断意图
端口号→攻击意图 (22=远程控制, 445=横向移动, 3306=数据窃取)
    ↓ 合成威胁评分
(攻击次数 × IP多样性 × 端口多样性) × 权重系数
    ↓ 生成告警并通知
```

**关键概念理解**:
- ✅ `response_ip` = **诱饵IP** (不存在的虚拟哨兵, 如 10.0.0.1)
- ✅ `attack_ip` = **被诱捕的内网失陷主机** (真实的被攻破设备, 如 192.168.1.100)
- ✅ `response_port` = **攻击者尝试的端口** (反映攻击意图, 如 3389=远程访问)
- ✅ 所有访问诱饵的行为都是**确认的恶意行为** (误报率极低)

**7个核心服务**:

1. **data-ingestion (8080)** ✅ 完成
   - 功能: 接收syslog日志 → 解析 → 发布Kafka
   - 特点: 高吞吐量(10000+ events/s), 失败重试, 连接池

2. **stream-processing (Flink)** ✅ 完成
   - 功能: 实时流处理 → 威胁聚合 → 发布事件
   - 特点: 3层时间窗口(30s/5min/15min), 增强威胁评分

3. **threat-assessment (8083)** ✅ 完成
   - 功能: 威胁评分计算 → 风险评估 → 建议缓解
   - 特点: 与原C#系统对齐, 多维度权重算法

4. **alert-management (8082)** ✅ 完成
   - 功能: 告警生成 → 多通道通知 → 生命周期管理
   - 特点: Email/SMS/Slack/Webhook/Teams, 智能去重

5. **customer-management (8084)** ✅ 完成
   - 功能: 客户CRUD → 设备绑定 → 通知配置
   - 特点: 多租户隔离, 订阅套餐管理

6. **api-gateway (8888)** ✅ 完成
   - 功能: 统一API入口 → 路由转发 → 安全控制
   - 特点: CORS支持, 认证/授权, 速率限制, 熔断降级

7. **config-server (8888)** ✅ 完成
   - 功能: 集中配置管理 → Git版本控制 → 属性加密
   - 特点: 动态配置更新, 多环境支持

### 第四层: 数据层 - 数据流与存储

**数据流**:
```
rsyslog (syslog端口9080)
    ↓ 原始日志
Data Ingestion Service
    ↓ 解析 + 序列化
Kafka: attack-events Topic
    ↓ 
Stream Processing (Flink)
    ↓ 30s窗口聚合
Kafka: minute-aggregations Topic
    ↓
Threat Assessment Service
    ↓ 威胁评分计算
Kafka: threat-alerts Topic
    ↓
Alert Management Service
    ↓ 多通道通知
PostgreSQL (威胁评估表 + 告警表)
```

**关键数据表**:
- `customers` - 客户信息 (customer_id, name, subscription_tier)
- `device_customer_mapping` - 设备-客户绑定 (dev_serial, customer_id)
- `customer_notification_configs` - 通知配置 (email_enabled, slack_webhook等)
- `threat_assessments` - 威胁评估结果 (threat_score, threat_level)
- `alerts` - 告警记录 (status, severity, description)
- `notifications` - 通知记录 (channel, recipient, status)
- `port_risk_configs` - 端口权重配置 (port_number, risk_weight)
- ... 12+其他业务表

  ```
````
